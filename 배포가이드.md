## Netlify 배포

### 방법 1: 드래그 앤 드롭 (가장 간단)

**소요 시간: 3분**

1. **Netlify 가입**
   - https://netlify.com 접속
   - "Sign up" 클릭
   - 이메일 또는 GitHub 계정으로 가입

2. **사이트 생성**
   - 로그인 후 "Add new site" 클릭
   - "Deploy manually" 선택

3. **파일 업로드**
   - 3개 파일(index.html, styles.css, script.js)을 폴더에 넣기
   - 해당 폴더를 드래그하여 Netlify에 드롭

4. **배포 완료**
   - 자동으로 URL 생성 (예: `random-name-123.netlify.app`)
   - 즉시 접속 가능

5. **도메인 연결 (선택)**
   - Site settings → Domain management
   - Custom domain 추가
   - DNS 설정 변경 (A 레코드)

### 방법 2: Git 연동 (자동 배포)

1. **GitHub 저장소 생성**
   ```bash
   git init
   git add index.html styles.css script.js
   git commit -m "Initial commit"
   git remote add origin [GitHub 저장소 URL]
   git push -u origin main
   ```

2. **Netlify 연동**
   - "Add new site" → "Import from Git"
   - GitHub 선택 및 저장소 연결
   - Build settings (정적 사이트이므로 설정 불필요)
   - "Deploy site" 클릭

3. **자동 배포**
   - 이후 Git push 시 자동 배포

## Vercel 배포

### CLI 방식

**소요 시간: 5분**

1. **Vercel CLI 설치**
   ```bash
   npm install -g vercel
   ```

2. **로그인**
   ```bash
   vercel login
   ```

3. **배포**
   ```bash
   cd [프로젝트 폴더]
   vercel
   ```

4. **프로덕션 배포**
   ```bash
   vercel --prod
   ```

### 웹 인터페이스 방식

1. https://vercel.com 가입
2. "Add New Project"
3. GitHub 저장소 연동 또는 파일 업로드
4. 자동 배포

## GitHub Pages 배포

### 요구사항
- GitHub 계정
- Git 기본 지식

### 배포 절차

1. **저장소 생성**
   - GitHub에서 새 저장소 생성
   - 저장소 이름: `cso-solution` (임의)

2. **파일 업로드**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/[username]/cso-solution.git
   git push -u origin main
   ```

3. **GitHub Pages 활성화**
   - 저장소 → Settings → Pages
   - Source: "Deploy from a branch"
   - Branch: main, folder: / (root)
   - Save

4. **접속**
   - URL: `https://[username].github.io/cso-solution/`
   - 배포 완료까지 1-2분 소요

### 커스텀 도메인 설정
```
Settings → Pages → Custom domain
→ 도메인 입력 (예: www.cso-solution.com)
→ DNS에 CNAME 레코드 추가
```

## AWS S3 배포

### 요구사항
- AWS 계정
- AWS CLI 설치 (선택)

### 배포 절차

#### 1. S3 버킷 생성

```bash
# AWS CLI 사용 시
aws s3 mb s3://cso-solution-website

# 또는 AWS Console에서:
# S3 → Create bucket
# Bucket name: cso-solution-website
# Region: ap-northeast-2 (서울)
# Block all public access: OFF (체크 해제)
```

#### 2. 정적 웹 호스팅 활성화

**AWS Console:**
1. 버킷 선택 → Properties
2. Static website hosting → Edit
3. Enable 선택
4. Index document: `index.html`
5. Save changes

#### 3. 버킷 정책 설정

**Policy 추가:**
```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::cso-solution-website/*"
        }
    ]
}
```

#### 4. 파일 업로드

**AWS CLI:**
```bash
aws s3 sync . s3://cso-solution-website
```

**AWS Console:**
- Upload 버튼 클릭
- 3개 파일 선택 및 업로드

#### 5. CloudFront 배포 (선택, CDN)

1. CloudFront → Create distribution
2. Origin domain: S3 버킷 선택
3. Default root object: `index.html`
4. Create distribution

#### 6. 도메인 연결 (Route 53)

1. Route 53 → Hosted zones
2. 도메인 선택
3. Create record
   - Record type: A
   - Alias: Yes
   - Route traffic to: CloudFront distribution

### 비용 산정
- S3 저장: $0.023/GB/월
- CloudFront 전송: 첫 10TB $0.085/GB
- **예상 비용**: 월 $1-5 (소규모 트래픽 기준)

## 전통적 서버 배포

### Ubuntu + Nginx 배포

#### 1. 서버 준비

```bash
# 서버 업데이트
sudo apt update && sudo apt upgrade -y

# Nginx 설치
sudo apt install nginx -y

# 방화벽 설정
sudo ufw allow 'Nginx Full'
sudo ufw enable
```

#### 2. 파일 배포

```bash
# 웹 루트 디렉토리로 이동
cd /var/www/html

# 기존 파일 백업
sudo mv index.nginx-debian.html index.nginx-debian.html.bak

# 파일 업로드 (SFTP 또는 SCP 사용)
# 로컬에서:
scp index.html styles.css script.js user@server-ip:/tmp/

# 서버에서:
sudo mv /tmp/*.html /tmp/*.css /tmp/*.js /var/www/html/
sudo chown www-data:www-data /var/www/html/*
sudo chmod 644 /var/www/html/*
```

#### 3. Nginx 설정

```bash
# 설정 파일 편집
sudo nano /etc/nginx/sites-available/default
```

**설정 내용:**
```nginx
server {
    listen 80;
    listen [::]:80;
    
    server_name cso-solution.com www.cso-solution.com;
    
    root /var/www/html;
    index index.html;
    
    location / {
        try_files $uri $uri/ =404;
    }
    
    # Gzip 압축
    gzip on;
    gzip_types text/css application/javascript;
    
    # 캐시 설정
    location ~* \.(css|js|jpg|png|gif|ico)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 4. SSL 인증서 설치 (Let's Encrypt)

```bash
# Certbot 설치
sudo apt install certbot python3-certbot-nginx -y

# SSL 인증서 발급
sudo certbot --nginx -d cso-solution.com -d www.cso-solution.com

# 자동 갱신 테스트
sudo certbot renew --dry-run
```

#### 5. Nginx 재시작

```bash
# 설정 테스트
sudo nginx -t

# 재시작
sudo systemctl restart nginx

# 상태 확인
sudo systemctl status nginx
```

### Apache 서버 배포

#### 1. Apache 설치

```bash
sudo apt update
sudo apt install apache2 -y

# 방화벽 설정
sudo ufw allow 'Apache Full'
```

#### 2. 파일 배포

```bash
cd /var/www/html
sudo rm index.html
# 파일 업로드 후
sudo chown www-data:www-data *
```

#### 3. Virtual Host 설정

```bash
sudo nano /etc/apache2/sites-available/cso-solution.conf
```

**설정 내용:**
```apache
<VirtualHost *:80>
    ServerName cso-solution.com
    ServerAlias www.cso-solution.com
    DocumentRoot /var/www/html
    
    <Directory /var/www/html>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
    
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
</VirtualHost>
```

#### 4. 사이트 활성화

```bash
sudo a2ensite cso-solution.conf
sudo a2dissite 000-default.conf
sudo systemctl restart apache2
```

#### 5. SSL 설치

```bash
sudo apt install certbot python3-certbot-apache
sudo certbot --apache -d cso-solution.com -d www.cso-solution.com
```

## 도메인 구매 및 연결

### 도메인 구매처

1. **가비아** (국내)
   - .com: 약 15,000원/년
   - .co.kr: 약 20,000원/년

2. **Namecheap** (해외)
   - .com: $10-12/년
   - 무료 WhoisGuard 제공

3. **Google Domains**
   - .com: $12/년
   - 간편한 DNS 관리

### DNS 설정

**Netlify/Vercel 연결:**
```
A 레코드
@ → Netlify IP (자동 제공)

CNAME 레코드
www → [사이트명].netlify.app
```

**AWS S3 + CloudFront:**
```
A 레코드 (Alias)
@ → CloudFront Distribution

CNAME 레코드
www → CloudFront Domain
```

## 문제 해결

### 일반적인 오류

**1. 404 Not Found**
- 파일명 확인 (대소문자 구분)
- 파일 권한 확인 (644)

**2. CSS가 적용되지 않음**
- 파일 경로 확인
- MIME 타입 확인

**3. HTTPS 오류**
- SSL 인증서 재발급
- Mixed Content 오류 확인

**4. 느린 로딩 속도**
- CDN 적용
- 이미지 최적화
- Gzip 압축 활성화
